    • Identify the technology as used by the cards using tools such as RFIDIOt and libnfc.
      Answer
      The cards use MIFARE Classic technology, which works according to ISO/IEC-14443A protocol, with NXP proprietary security protocol (Crypto-1) for authentication and encryption.

        $ mfoc -O card.mfd
  Found Mifare Classic 1k tag
  ISO/IEC 14443A (106 kbps) target:
  <...>

      The MIFARE Classic 1K offers 1,024 bytes of data storage, split into 16 sectors of 4 data blocks each; each sector is protected by two different keys, called A and B.

        The file shows all 16 sectors. Here is an example of one sector: 3x16 bytes of data followed by 16 bytes of access keys and accecss bits:

         Block 1 | ea e3 68 ee 8f 08 04 00    01 a9 aa 13 90 e5 f1 1d
 Block 2 | 00 01 02 03 04 05 06 07    08 09 0a 0b 0c 0d 0e 0f
 Block 3 | 00 00 00 00 00 00 00 00    00 00 00 00 00 00 00 00
 Block 4 | ff ff ff ff ff ff ff 07    80 69 ff ff ff ff ff ff

        The block 0 of sector 0 contains special data. The first 4 data bytes contain the unique identifier of the card (UID)

         ea e3 68 ee

        followed by its 1-byte bit count check (BCC).

         8f

        The bit count check is calculated by successively XOR-ing all UID bytes.

        The remaining bytes are used to store manufacturer data.

         08 04 00    01 a9 aa 13 90 e5 f1 1d

        This data block is read-only (but there are UID rewritable changeable Mifare Classic cards).

         Block 1 | ea e3 68 ee 8f 08 04 00    01 a9 aa 13 90 e5 f1 1d

        The following two blocks contain some data:

          Block 1 | 00 01 02 03 04 05 06 07    08 09 0a 0b 0c 0d 0e 0f
  Block 2 | 00 00 00 00 00 00 00 00    00 00 00 00 00 00 00 00

        The last block of each sector is called sector trailer

          Block 4 | ff ff ff ff ff ff ff 07    80 69 ff ff ff ff ff ff

        It contains the secret keys A and B (each 6 bytes) which are used for authentication

         Key A | ff ff ff ff ff ff
 Key B | ff ff ff ff ff ff

        the access conditions (3 bytes) to define which operations are available for this sector

         AC | ff 07 80

        and one data byte (U) remaining which has no defined purpose

         U | 69

  2. Oscilloscope analyzer:
    Get the magic lock device:

    Oscilloscope

    • Can you find out what protocol is used?
      Answer
      For data transfer the NRZ code is used which is not self-synchronizing, so the start and stop bits are used for synchronization, allowing to select the bit sequence and synchronize the receiver with the transmitter.

    • Can you retrieve the secret code algorithm using sniffing?
      Answer
      The length of the transmitted data is 2 bytes, but the oscilloscope shows only 8 bits (the right half of the transmitted data). We already know that the encryption algorithm uses a shifting. A special formula is used to obtain the magic number. This formula contains a few bits from the left half which XORed with 0 or 1. Having accumulated a large amount of data, we can find a regularity and calculate the magic number:

        | x7 | x6 | x5 | x4 | x3 | x2 | x1 | n |
  
  n = x11 + x9 + 1

      where xk - a bit on the k position, n - magic number.

      In addition, the noise was added to the data:

        | x7 | x6 | x5 | x4 | x3 | x2 | x1 | n  |
  xor
  | 1  | 1  | 0  | 0  | 1  | 1  | 0  | 0  |

      After that, the number was shown on the oscilloscope.

      Also, we can use the following python code to bruteforce the magical number that has been XORed with the shifted value

        n = 0xd3
  l = 0x7f
  for shift in range(254):
      now = n ^ shift
      next = l ^ shift
      if bin(now)[3:] == bin(next)[2:-1]:
          print(hex(shift))
          print(bin(now))
          print(bin(next))
          print("------------------")

    • Are there any other weaknesses?
      Answer
      This algoritm is impractical in most realistic scenarios. Does not guarantee integrity, attacker can easily change it to something else. Insecure if keys are reused, attacker can obtain XOR of plaintexts.

    • How can you protect against attacks?
      Answer
      Use more complicated algorithm to generate pseudo-random number for magic number.
