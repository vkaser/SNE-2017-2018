Workshop 3 - Memory analysis

Situation:

You have a memory image taken from a laptop, which was suspected to be involved in a serious e-banking incident. It is to be assumed that some malware could be on it. You suspect it could be either Zeus or Gozi, as these two malware families are currently active in your country.

Make yourself familiar with the malware families and what characteristics they have and how they can potentially be recognized.

It’s now your job to find out what could have happened.

  • A memory image from the PC was drawn and is stored on the VMware image
  • The registry was dumped too and is also available for investigation.

    • What information can be found?

        $ strings images/MEMORY-IMG2.DMP > strings.txt

  $ grep -E '*\.dll' strings.txt > dllfiles.txt

Scalpel

Although strings reveals a lot of information, processing large images can be cumbersome. Today’s systems can easily have 4-32GB RAM or even more. This means an even more powerful system is required for the analysis.

  • Try to complete/confirm your textual information found above, by carving for files using scalpel:

    • Familiarise yourself with scalpel
      scalpel is an open source program for recovering deleted files from a disk image or raw block device based on headers and footers specified by the user.

    • Look at the config in Exercises/scalpel.conf
      The configuration file is used to control the types of files Scalpel will attempt to carve. For each file type, the configuration file describes the file’s extension, whether the header and footer are case sensitive, the maximum file size, and the header and footer for the file. The footer field is optional, but the header, size, case sensitivity, and extension fields are required.

    • What information does scalpel reveal?
      Information that has been deleted and corresponds to the scalpel configuration file. By default in the config file, there are some strings which are commented. Let’s uncomment these strings and perform the following command:

        scalpel -o files -c scalpel.conf images/MEMORY-IMG2.DMP

Additional Question to Answer:
Imagine the situation, there was an incident on PC(Windows) but the suspected PC(Windows) is locked. What can you do to be able to produce live forensics? What challenges do you have?

Answer:
In case of the locked PC, it is possible to produce the FireWire attack via Firewire serial bus. This attack allows gaining read/write access to a machine’s memory by means of Direct Memory Access (DMA). As this is DMA, the exploit works regardless of whether the target PC is locked or even logged on. The technique requires that the computer has a FireWire port and working FireWire drivers are installed.
In addition, we are able to produce Cold Boot Attack, which involve copying of the system memory once the system has been powered off. To execute this attack, a running computer is cold-booted (hard reboot). A removable disk is then immediately used to boot a lightweight operating system, which is then used to dump the contents of pre-boot physical memory to a file. Alternatively, the memory modules are removed from the original system and quickly placed in a compatible machine under the attacker’s control, which is then booted to access the memory.
