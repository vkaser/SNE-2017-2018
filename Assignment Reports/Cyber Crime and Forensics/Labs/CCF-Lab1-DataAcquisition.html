1 Imaging

  1. Read about DEFT Linux and its features.
    a) Why would you use a Forensic distribution and what are the main differences between a regular distribution?
    Answer
    Because this distribution contains a sufficient set of open source tools for Computer Forensics. The main difference between a regular distribution is a highly specialized field of application. It aims to help with Incident Response, Cyber Intelligence, and Computer Forensics scenarios. Amongst others, it contains tools for Mobile Forensics, Network Forensics, Data Recovery, and Hashing.

    b) When would you use a live environment and when would you use an installed environment?
    Answer
    A live environment is used when you can not access the system (for example, the password for logging on) or when there is a possibility that the data can be destroyed when the system boots. Therefore, in the live environment, you can make an image of the information carrier for further analysis in the installed environment.
    Also live environment should be used when there is a special hardware that we do not have (for example RAID controller), In this case, live environment is a must.
    Installed environment can be used after taking an image (clone) of the media that we want to investigate.

    c) What are the priorities of DEFT? And how does it compare to other forensic distributions?
    Answer
    DEFT stands for Digital Evidence & Forensics Toolkit. The main priority of this distribution is to create a set of utilities that would be sufficient for computer forensics and to provide the necessary level of security in the analysis. This distribution was created by people involved in the implementation and development of applications which are released to Law Enforcement Officers. There are different forensic distributions: SIFT, Caine, Kali, Paladin, Helix or DEFT. But, Deft features the DART suite and there is no equivalent in the Unix world. DART is an application that organizes, collects and runs software in safe mode for the purpose of live forensic analysis and incident response. In addition, this distribution is trusted by the military, police, private security professionals and IT auditors.

3 Live Forensics

Procedures and preferences change during the live forensic acquisition. Take for example memory into account. This is one of the most valuable resources.

  1. What kind of things would be less important during the live acquisition?
    Answer
    A live acquisition is an analysis that is done on a live system. Drive encryption would be less important during live acquisition.

  2. What would be different in your method?
    Answer

  • Volatile Data: This includes all the data that can be changed or erase in case we change or install anything on the suspect system, for example, active network connections, logged-in users, running processes on the suspect system, scheduled tasks and finally clipboard contents.
  • Non-Volatile Data: This includes any data that will be the same even in case we change few things in the suspect system, for example, System logs, browsers history, page files and finally disk content.
  1. Describe the new method that you would use to gather data during live forensics. Make sure to categorize by priority.
    Answer
Category Items
Memory Memory dump
Processes User process and Kernel process
Network activity IP address, MAC address, Routing table, ARP table, Active network connections, Remotely logged in users,
Basic data System time, Network connection, Disk partitions, User accounts, Installed hardware
System data OS name, OS version, Uptime, list of installed application, Registry (in case of windows)
User data Web browsers history and cookies, downloaded files, Temporary directory, Trash bin items, system logs, History of commands (in case of Linux)

  3. Create a timeline on the created image. Use the appropriate tools for this.
    There are two main commands to generate timelines of the filesystem:

    • fls takes as input a raw file derived from the acquisition of a mass memory (either a single partition either a raw image of a disk with multiple partitions), and returns a list of all files (allocated or not whose, however, the record is still contained in the file allocation table), for subsequent use with mac-time;

        deft8 /mnt/suspect % fls -o 8064 -z GMT -s 0 -m C: -f fat -r ../lenovo/T2-disk-imaging-part1-suspect-usb-key.dd > ../lenovo/harddisk-c.body

    • mactime takes as input a list, created by fls, with all the data contained in the file system under analysis and provides a timeline in ASCII format.

        deft8 /mnt/suspect % mactime -b ../lenovo/harddisk-c.body -z GMT -d > ../lenovo/disk.csv

      In the csv-file we found a correspondence in the user’s Skype with the name Mallory.Evil, who asked to send him the book Star Wars.
